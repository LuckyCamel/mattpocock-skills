# caveman（极简沟通模式）

## 用途

在保持技术准确性的前提下，用极度压缩的方式回复，减少约 75% 的「废话」token：去掉冠词、填充词、客套与过度委婉，适合明确说「caveman mode」「少 token」「极简说」或调用 `/caveman` 时使用。

## 行为要点

- **持久**：一旦触发，后续每轮回复都保持该模式；仅当用户说「stop caveman」或「normal mode」时退出。
- **规则**：可省略 a/an/the、just/really 等填充词；技术术语与代码块保持原样；可用箭头表示因果（X → Y）。
- **安全例外**：安全警告、不可逆操作确认、易被误读的复杂步骤、用户要求澄清时，暂时恢复正常表述，完成后再恢复 caveman。
