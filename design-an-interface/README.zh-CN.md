# design-an-interface（接口设计：设计两次）

## 用途

受《A Philosophy of Software Design》中「Design It Twice」启发：对同一模块并行生成**多种差异很大**的接口设计方案，再对比取舍。适用于设计 API、比较模块边界、或用户提到「design it twice」等场景。

## 工作流概要

1. **收集需求**：问题、调用方、关键操作、约束、对外暴露与对内隐藏。
2. **并行子代理**：用 Task 工具同时派 3+ 个子代理，每个带不同约束（如最少方法数、最大灵活性、最常见路径优化、某范式启发）。
3. **展示**：签名、用法示例、内部隐藏复杂度。
4. **对比**：简洁性、通用 vs 专用、实现效率、深度、易用 vs 易误用。
5. **综合**：询问用户主场景及是否融合多方案优点。

## 注意

只做接口形态讨论，不实现；强制子方案之间要**真正不同**。
