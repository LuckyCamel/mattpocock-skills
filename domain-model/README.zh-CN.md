# domain-model（领域模型拷问与文档）

## 用途

围绕你的方案做**高强度一对一问答**，沿决策树逐分支澄清；结合仓库已有领域语言与挑战术语，并在讨论过程中**就地更新** `CONTEXT.md`、按需撰写 ADR。适合要把计划与项目「统一语言」和已记录决策对齐时使用。

## 行为要点

- 一次只问一个问题，等待回答；能靠读代码回答则先探索代码。
- 对照 `CONTEXT.md` 术语表指出冲突；把模糊词（如 account）收窄为规范概念。
- 用具体场景 stress-test 领域关系；声称与代码不符时指出矛盾。
- **CONTEXT.md**：术语定下来就立即更新，格式见 [CONTEXT-FORMAT.md](./CONTEXT-FORMAT.md)。
- **ADR**：仅当「难回滚 + 无上下文会惊讶 + 真实权衡」三者齐备时再提议，格式见 [ADR-FORMAT.md](./ADR-FORMAT.md)。
