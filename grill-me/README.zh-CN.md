# grill-me（计划/设计拷问）

## 用途

就当前方案**持续追问**，直到双方对设计达成共识：沿决策树逐分支理清依赖；每个问题附带**推荐答案**；**一次只问一个问题**。适合 stress-test 计划、被「拷问」设计、或用户明确说 grill me。

## 行为要点

若问题可通过阅读代码解决，则优先探索代码而非追问用户。

## 与 domain-model 的区别

`grill-me` 是通用拷问流程；`domain-model` 额外绑定 `CONTEXT.md` / ADR 更新与领域术语纪律。
